### 2.1 算法的定义

* **算法**：解决问题方法的描述，在计算机中描述为有限的指令序列，每条指令表示一个或多个操作，完成特定的功能
* **算法的特性：有穷性、确定性、可行性、有输入输出**



**何为好算法？**

* **正确性**（最基本的要求）
* **可读性**（便于理解交流）
* **健壮性**（能够处理不合法的输入）
* **高效率**（在输入很大时运行速度快）
* **低存储量**（相比于其他算法占用更小的空间）



### 2.2 算法效率的度量



#### 2.2.1 事后统计

利用设计好的测试程序和数据以及计算机的计时，对不同算法执行时间进行统计和比较。但这受到机器性能和编程语言的影响。由于事后统计的特性，必须等待程序执行完成才能得知其性能，而有些算法无法进行事后统计。



#### 2.2.2 事前分析估算

根据**统计方法**对算法进行估算，**考虑时间开销$T(n)$与输入规模$n$的关系**

按照这个规则来分析下面的累加代码：

```C++
#include<iostream>
using namespace std;
int main()
{
    int s = 0;						// +1
    for(int i = 1; i < 10001; i++)	// +10000
        s += i;						// +10000
    cout << s << endl;				// +1
    return 0;
}
```

可以得出$T(n) = 2n + 2$，但这样写起来仍然复杂，我们使用渐进分析的概念，来定义算法的**时间复杂度**，也就是熟知的大$O$表示法（big-oh notation）：如果一个算法的时间复杂度为$O(f(n))$（自然地，$\lim\limits_{n \to \infty} f(n) = \infty$），那么其时间开销函数满足
$$
\lim_{n \to \infty} \frac{T(n)}{f(n)} = C,~~ C \in \R_{++}
$$
通过极限的知识，不难证明其以下几个性质：

* **忽略常量**：$O(f(n)+C) =O(f(n))$
* **只考虑更高阶的部分**：$O(n^3 + n^2) = O(n^3)$
* **加法规则：**$O(f(n)) + O(g(n)) = O(f(n))$，如果$f(n)$更高阶
* **乘法规则**：$O(f(n))O(g(n)) = O(f(n)g(n))$

Remark. 我们在计算算法的时间复杂度时，常常计算最坏的情况。在一些有随机因素的算法中，我们常考虑其消耗时间期望的复杂度（如快速排序）



从空间的角度来看，就也有**空间复杂度**。它指征算法运行时所需空间的大小。往往高效率和低存储消耗不可兼得，所以也就有了**空间换时间**的说法。很典型的例子是递归和数据对齐。
